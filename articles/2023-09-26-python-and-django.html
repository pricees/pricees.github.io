Types The first thing that I noticed is that Python requires you to know a few more types: sets, lists, dictionaries, and tuples. It forces you to know these by using them it self for args and kwargs, returns from Django libraries etc. And while Ruby has sets, the reality is that most of the time, in Ruby, ones togo data structures are List and Dictionary. I dare say the average rubyist does not know the difference between a Tuple and a Lsit/Array, let alone other data structures, and why should we? One Way To Do Everything I love Ruby and like it's expressiveness. Python has, essentially one way to do each thing. What is interesting is that if you are using a Ruby linter like Rubocop or have many authors of the same code base, my gut is that the artistic and expressive nature of Ruby with regress to the most practical and intuitive Closures, Lamddas, Functional Programming I like Ruby's syntax for these better and when one gets bit by the functional bug Ruby really shines for it's express declarative chaining.
